# 제로스트 · 마녀스프 MVP 기획서

> Notion에 붙여넣기용. 최종 업데이트: 2026-05

## 0. FE / BE 역할 (팀 합의)

| 영역 | BE | FE (이 레포) |
|------|-----|----------------|
| 출석·랜덤 재료 | 1일 1회 검증, 확률, 인벤토리 저장 | `POST /check-in` 호출, 응답 UI 반영 |
| 미션 검수·재료 지급 | 인증·지급 (추후) | 지금은 데모 플로우 + mock |
| 스프 제작·레시피 | 조합 검증·1회 제한 (추후) | 슬롯 UI + mock |
| 에코잼·실물 리워드 | 원장·지급 (추후) | 결과 화면 표시 |

**FE는 비즈니스 규칙을 “결정”하지 않고, API 계약에 맞춰 화면만 만든다.**

## 1. 한 줄 컨셉

일상 **제로 미션**을 하면 **재료**가 쌓이고, 냄비에 재료를 넣어 **마녀스프**를 끓이면 **에코잼(인게임)** 또는 **실물 리워드(히든)** 를 얻는다.

## 2. 목표

- 기능 최소 · 체감 가벼움
- **리워드는 화면에서 확실히** (에코잼 / 실물)
- 주간 레시피 + 히든 1회 제작으로 **재방문**

## 3. 하단 네비 (4탭)

| 탭 | 역할 |
|---|---|
| 제작 | 보유 재료 · **슬롯(일반 3 + 히든 1)** · 스프 만들기 |
| 홈 | 마녀(고양이) + 냄비 · **오늘의 레시피 힌트** · 미션 진입 |
| 레시피 | 주간 일반(공개) · 히든(??? ) |
| 마이 | 에코잼 · 완성 레시피 · 실천 기록 |

## 4. 제작 UX

```
[ □ ] [ □ ] [ □ ] | [ □ ]   ← 일반 3칸 + 히든 1칸 (UI 4칸)
─────────────────────
(보유 재료 그리드, 탭 시 빈 칸에 순서대로 채움)
칸 탭 → 해당 재료 제거
─────────────────────
[ 스프 만들기 ]  ← 재료 3개(일반) 또는 4개(히든)일 때 활성
```

- 보유 수량 0인 재료는 비활성
- **일반(이번주) 레시피:** 재료 **3개** (앞 3칸만 사용)
- **히든 레시피:** 재료 **4개** (네 칸 모두)
- 조합 일치 시 제작 (검증은 **BE**)
- **각 레시피는 1회만** 제작 가능

## 5. 레시피 규칙

| 구분 | 재료 수 | 변경 주기 | 노출 | 보상 |
|---|---|---|---|---|
| **이번주 레시피** | **3** | 매주 | 공개 | **에코잼** |
| **히든 레시피** | **4** | 고정 | 미완료 `???` | **실물** |

## 6. 홈 (마녀스프)

- 중앙: 마녀(또는 마녀 고양이) + 냄비(저어지는 연출은 Phase 2)
- **오늘의 레시피 힌트** 1줄
- CTA: **오늘 미션 하기** → 기존 미션 플로우

## 7. 출석 · 미션 → 재료

### 출석 (홈 카드 1탭) — **BE 담당**
- `POST /check-in`: 하루 1회 검증, **랜덤 재료 1개** 지급, `streakDays`·`ingredientInventory` 반환
- FE: 카드 탭 → API 호출 → 응답만 상태에 반영 (랜덤·중복 검증은 FE에서 하지 않음)
- MVP 로컬: `src/api/mock/checkInMock.ts` 스텁만 사용
- 별도 화면·광고·푸시 없음 — 토스트앱에 맞게 최소 UX

### 미션
- 미션 완료 시 **재료 1종** 지급 (미션별 고정 매핑)
- 결과 화면에 「○○ 재료를 받았어요」

## 8. 스프 만들기 결과

- 성공: 연출 + 레시피명 + **에코잼 +N** 또는 **실물 리워드 안내**
- 이미 만든 레시피 조합 → 「이미 만든 스프예요」

## 9. 비주얼

- 포인트 컬러: **보라 계열** (`#7C5CBF`, `#EDE7F6`)
- 톤: 앱인토스 해요체, 카드·여백 넉넉히

## 9. 온보딩

- 플로우 유지: 환영 → 샵 선택
- 비주얼만 보라 톤 + 마녀스프 세계관 카피
